    学习总结

    建议在学习这部分内容要有扎实的基础，和一定的代码沉淀。

    jQuery

    他为我们提供了简便的DOM操作，以及对不同浏览器的兼容性，用面向对象的思想我们可以任意扩展我们自定的方法保存到jQuery原型对象中去。

    Underscore

    其实这部分内容增强了JS语言的表现力，js内置的对象并不能完全解决我们日常工作中所有的问题而我们可以扩展一些属性和方法来解决日常的问题,Underscore将扩展的东西以流式的思想将其贯穿起来,就像是一个精致的管道工程,来解决一些数据源的问题，从哪里来->要经过哪里->最后到哪里去。

    Modularization

    模块化的思想可以为我们实际工作解决很多问题，把一个系统分割成独立功能的方法，这样不仅降低了维护成本,解决了文件依赖曹诚的排错困难,还提升了生产效率,职责更明确,思路更清晰。

    Gather

    集合三者的优势及特点构建自己的前端业务架构,三者之间各有利弊,可以贯穿三者的优势和思路去搭建一个原生JS框架

    extend源码

    可以给任意对象扩展，也可以给jQuery本身扩展。 扩展extend 通过参数判断 利用arguments拿到当前对象的参数列表 这里要注意的是 extend方法中区分了 浅拷贝和深拷贝区别: 浅拷贝 如果要两个对象中存在相同的对象名称 最终的会进行覆盖. 深拷贝 会将两个相同名称的对象进行合并成一个对象 每个对象中的属性并列存在.

      (function(root){
  var jQuery = function(){
      // return new jQuery ();
      //这样会不断的创建对象不断的调用构造函数。进入死循环状态
      return new jQuery.prototype.init();
    }
  jQuery.fn = jQuery.prototype = {
      init: function(){
          
      },
      css: function(){
          alert("This`s CSS")
      }
  }
  jQuery.fn.extend = jQuery.extend = function(){
    
    var target = arguments[0] || {};
    var length = arguments.length;
    var i = 1;
    var deep = false;
    var option,name,copy,src,copyIsArray,clone;
    if(typeof target === "boolean"){
        deep = target;
        target = arguments[1];
        i = 2;
    }
    if(typeof target !== "object"){
        target = {};
    }
    //参数的个数 1
    if(length === i){
        target = this;
        //this的指向如果是jQuery调用指向本身
        //实例化对象调用时指向实例化对象
        i--;
    }
    //浅拷贝 (深拷贝的特点会有一个boolean值在最前面传进来)
    for(;i<length;i++){
        if((option = arguments[i]) != null){
            for(name in option){
                // console.log(name)
                // target[name] = option[name];
                copy = option[name];
                src = target[name];
                if(deep && (jQuery.isPlainObject(copy) || (copyIsArray = jQuery.isArray(copy)))){
                    if(copyIsArray){
                        copyIsArray = false;
                        clone = src && jQuery.isArray(src) ? src : [];
                    }else{
                        clone = src && jQuery.isPlainObject(src) ? src : {};
                    }
                    target[name] = jQuery.extend(deep,clone,copy);
                }else if(copy != undefined){
                    target[name] = copy;
                }
            }
        }
    }
    return target;
  }
  jQuery.fn.init.prototype =  jQuery.fn;

    jQuery.extend({
    // 类型检测
    isPlainObject: function(obj){
        return toString.call(obj) === "[object Object]";
    },
    isArray: function(obj){
        return toString.call(obj) === "[object Array]";
    }
    });
    
  root.$ = root.jQuery = jQuery;
  })(this)

    DOM对象：

    -是由宿主环境提供的的对象，在文档对象模型中，每一个部分都是节点。

    -例：所有HTML元素是元素节点，而Element 对象表示HTML元素。

    -HTML DOM Element 对象可以拥有类型为元素节点||文本节点||注释节点 的子节点。

    jQuery对象

    -jQuery函数创建出来的对象。

    -通过jQuery选择器可以获取HTML元素，并且以一种数组形式存储在jQuery对象中

    jQuery选择器接口:

    1.传入对象 $(this) $(document)这样就把传入的对象包装成了jQuery对象 注意※ 当然你直接在script中使用$(this)时 this指向的是window

    2.传入函数 $(function(){}) 这个是在页面DOM文档加载完成之后执行的，等效于在DOM加载完毕后执行$(document).ready()方法。为什么用document是指当前window底下的document,换句话说DOM树加载完毕意味着可以操作DOM了就酱紫。

    3.传入字符串$('.box')查询DOM节点包装成jQuery对象 就是将所有class="box"DOM标签包装到了jQuery对象中

    4.传入HTML字符串 $('<div>')

    如果是查询他会把dom中所有的div包装成 $('<div>')这个对象

    如果是创建DOM节点 他就会创建并包装成jQuery对象

    5.为空$()他就是一jQuery的实例对象
